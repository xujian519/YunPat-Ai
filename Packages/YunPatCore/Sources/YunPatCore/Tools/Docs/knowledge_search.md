# knowledge_search

## 适用场景

- 需要引用具体法条（如专利法第22条第3款）作为审查意见论据
- 检索审查指南对特定程序的规定（如优先审查、分案申请规则）
- 查找同类案件判例以支持创造性或无效论证

## 典型流程

1. 将用户提问提炼为精确的法律术语或条文编号作为 `query`
2. 若用户限定了知识类型，设置 `category`（`regulation` / `guideline` / `case`）
3. 调用工具，解析 `results` 数组，选取 `relevance` 最高的条目引用

## 参数

| 参数       | 类型   | 必填 | 默认 | 说明                                      |
| ---------- | ------ | ---- | ---- | ----------------------------------------- |
| `query`    | string | 是   | —    | 检索词，推荐使用法条编号或精确法律术语    |
| `category` | string | 否   | —    | 限定类型：`regulation` / `guideline` / `case` |
| `limit`    | int    | 否   | `10` | 返回结果数量上限                          |

## 返回值

成功（`ok: true`）：

```json
{
  "ok": true,
  "data": {
    "query": "专利法第22条第3款 创造性",
    "results": [{ "title": "专利法第22条", "source": "专利法", "snippet": "……", "relevance": 0.95 }]
  }
}
```

失败（`ok: false`）：
```json
{ "ok": false, "error": { "code": "INVALID_ARGS", "message": "query 参数为必填项", "hint": "请提供非空检索词后重试" } }
```

## 错误码

| 错误码         | 含义               | 恢复方式                           |
| -------------- | ------------------ | ---------------------------------- |
| `INVALID_ARGS` | `query` 为空或缺失 | 提供非空检索词后重新调用           |
| `NO_RESULTS`   | 未匹配到任何结果   | 放宽检索词、移除 `category` 限制   |

## 技巧

- 法条查询优先使用条文编号（如 `"专利法第22条第3款"`），比口语化表述更精准
- 法规与指南检索宜用精准法律术语，避免口语化（如用"优先权恢复"而非"补交优先权"）
- 结果按 `relevance` 降序排列，优先使用前两条；不确定 `category` 时留空以获取跨类型结果

## 已知限制

- 覆盖中国专利法律体系（含实施细则、审查指南），不含外国法
- 当前为 stub 实现，正式版接入宝宸知识库后端服务
